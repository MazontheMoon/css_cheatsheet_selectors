Simple Selectors

The simplest of them all! The style is applied to one or more elements by targeting:

  • Elements
  • Classes
  • ID
  • Universal
Elements

The style is applied to every element of that type and is useful for applying page wide or fallback styles

/*All p elements are blue*/
p {
color: blue
}

Classes

The class selector consists of a dot (.) followed by a class name (with no spaces in the class name).
Multiple elements in the document can have the same class name and a single element can have multiple class names (separated by spaces) which makes this selector type useful when applying a style to a variety of different types of elements.

/*Add a solid black border*/
.box-outline {
border: 2px solid black;
}

ID

The ID selector consists of a hashtag (#) followed by ID name (must be unique) of a given element.
This is different to class styling as multiple html elements can use the same class but only a single element can have an id (There can be only one!)

/*styling for first box in form*/
#top-box {
background-color: grey;
border: 2px dashed black;
}

Universal

The most simple selector of them all may be the universal selector.
* is the universal selector and is rarely useful by itself but often used in combination with other selectors.

/*reset default margins*/
ul * {
margin: 0;
padding: 0;
border: 1px dashed grey;
}

Attribute Selectors

Attribute selectors match one or more elements based on their attributes/attribute values.

The generic syntax consists of square brackets [] containing an attribute name followed by optional condition to match against the value of the attribute (this makes much more sense when you see the examples below).

These can be split into two categories:

  • Presence and Value
  • Substring value
Presence and Value

Try to match an exact attribute or attribute value of an element. This will make much more sense when you see the examples below (Well...a bit more sense)

  • [attr] – this will select all elements with the attribute attr

    /*style all li elements with a class attribute*/
    li[class] {
    list-style: none;
    }

  • [attr=val] – this will select all elements with the attribute attr but only if attr value is val

    /*style li elements with a class attribute with the value "classy"*/
    li[class="classy"] {
    background-color: grey;
    }

  • [attr~=val] – this will select all elements with the attribute attr but only if the value 'val' is one of a space separated list of values for attr

    /*style li elements with a class attribute with a value of “class” but only when “class” is not the only value for that attribute - say that five times fast*/
    li[class~="class"] {
    color: purple;
    }

Substring value

The next level to presence and value, this offers flexible matching in a similar fashion to regular expression (also known as RegExp-like selectors)

  • [attr|=val] – this will select elements with the attribute attr for which the value is exactly 'val' or starts with val

    /*style for all li elements with a class attribute but only if the value is "class"*/
    li[class|="class"] {
    font-size: 2rem;
    }

  • [attr^=val] – this will select all elements with the attribute attr for which the value starts with 'val'

    /*style for li elements with a class attribute where the value begins with "class"*/
    li[class^="class"] {
    color: purple;
    }

  • [attr$=val] – this will select all elements with the attribute attr for which the value ends with 'val'

    /*style for li elements with a class attribute where the value ends with "class"*/
    li[class$="class"] {
    color: purple;
    }

  • [attr*=val] – this will select all elements with the attribute attr for which the value contains the string 'val' (unlike [attr~=val] it does not treat spaces as value separators but as part of attribute name)

    /*style for li elements with a class attribute where the value includes "class" anywhere in the string*/
    li[class*="class"] {
    text-decoration: underline;
    }

Pseudo Class

Pseudo-classes allow you to select one or more elements that exist in a certain state such as mouse hover, or an element that is the first child of its parent in the DOM tree (like the first word in a paragraph, or the first div in an article, etc.)

A pseudo-class is a keyword preceded by a colon(:) added to the end of selectors to specify you want to style the selected elements but only when in a certain state such as mouse hover.

  • /*Change color of link on mouse hover*/
    a:hover {
    color: purple;
    }

Pseudo Element

Pseudo-elements may be used more often than pseudo-classes and are used to target elements that do not technically exist in the HTML document in that they allow you to add content to an element in the HTML document.

Similar to pseudo-classes but are preceded with two colons (::) and are added to the end of selectors to select a certain part of the element.

You can use only one pseudo-element in a selector. It must appear after the simple selectors in the statement.

  • /*add arrow icon after all external http links*/
    [href^=http]::after {
    content: '⤴';
    }

  • Some examples...

    • ::after
    • ::before
    • ::cue
    • ::cue-region
    • ::first-letter
    • ::first-line
    • ::selection
    • ::slotted()

Combinators

Not exactly selectors themselves but ways of combining two or more selectors in useful ways for very specific selections, such as only paragraphs that come directly after headings

CSS has several ways to select elements based on how they are related to one another:

  • Group of selectors (A, B) - Any element matching A and /or B

    /*All img elements within li elements and/or all a elements within li elements*/
    li img, li a {
    border: 2px solid red;
    }

  • Descendant selector (A B) - Any element matching B which is a descendant of an element matching A

    /*Any a elements within a li element*/
    li a {
    border: 1px dotted black;
    }

  • Child selector (A > B) - Any element matching B that is a direct child of an element matching A

    /*Any ul element within a li element*/
    li > ul {
    background-color: grey;
    }

  • Adjacent sibling selector (A + B) - Any element matching B that is the next sibling of an element matching A

    /*Any p element that comes immediately after a h1 element*/
    h1 + p {
    font-size: 1.5rem;
    }

  • General sibling selector (A ~ B) - Any element matching B that is one of the next siblings of an element matching A

    /*All p elements that come after a h1 element*/
    h1 ~ p {
    color: darkgrey;
    }

Multiple Selectors

Not separate selectors; the idea is that you can put multiple selectors on the same CSS rule, separated by commas, to apply a single set of declarations to all the elements selected by those selectors

  • /*fallback font style*/
    h1, h2, h3, h4, h5, h6, p, li, a {
    font: italic normal bold normal 1em/1.5 Helvetica, Arial, sans-serif;
    }
