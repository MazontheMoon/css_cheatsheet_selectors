Simple Selectors

The simplest of them all! The style is applied to one or more elements by targeting:

  • Elements
  • Classes
  • ID
  • Universal
Elements

The style is applied to every element of that type and is useful for applying page wide or fallback styles

/*All p elements are blue*/
p {
color: blue
}

Classes

The class selector consists of a dot (.) followed by a class name (with no spaces in the class name).
Multiple elements in the document can have the same class name and a single element can have multiple class names (separated by spaces) which makes this selector type useful when applying a style to a variety of different types of elements.

/*Add a solid black border*/
.box-outline {
border: 2px solid black;
}

ID

The ID selector consists of a hashtag (#) followed by ID name (must be unique) of a given element.
This is different to class styling as multiple html elements can use the same class but only a single element can have an id (There can be only one!)

/*styling for first box in form*/
#top-box {
background-color: grey;
border: 2px dashed black;
}

Universal

The most simple selector of them all may be the universal selector.
* is the universal selector and is rarely useful by itself but often used in combination with other selectors.

/*reset default margins*/
ul * {
margin: 0;
padding: 0;
border: 1px dashed grey;
}

Attribute Selectors

Attribute selectors match one or more elements based on their attributes/attribute values.

The generic syntax consists of square brackets [] containing an attribute name followed by optional condition to match against the value of the attribute (this makes much more sense when you see the examples below).

These can be split into two categories:

  • Presence and Value
  • Substring value
Presence and Value

Try to match an exact attribute or attribute value of an element. This will make much more sense when you see the examples below (Well...a bit more sense)

  • [attr] – this will select all elements with the attribute attr

    /*style all li elements with a class attribute*/
    li[class] {
    list-style: none;
    }

  • [attr=val] – this will select all elements with the attribute attr but only if attr value is val

    /*style li elements with a class attribute with the value "classy"*/
    li[class="classy"] {
    background-color: grey;
    }

  • [attr~=val] – this will select all elements with the attribute attr but only if the value 'val' is one of a space separated list of values for attr

    /*style li elements with a class attribute with a value of “class” but only when “class” is not the only value for that attribute - say that five times fast*/
    li[class~="class"] {
    color: purple;
    }

Substring value

The next level to presence and value, this offers flexible matching in a similar fashion to regular expression (also known as RegExp-like selectors)

  • [attr|=val] – this will select elements with the attribute attr for which the value is exactly 'val' or starts with val

    /*style for all li elements with a class attribute but only if the value is "class"*/
    li[class|="class"] {
    font-size: 2rem;
    }

  • [attr^=val] – this will select all elements with the attribute attr for which the value starts with 'val'

    /*style for li elements with a class attribute where the value begins with "class"*/
    li[class^="class"] {
    color: purple;
    }

  • [attr$=val] – this will select all elements with the attribute attr for which the value ends with 'val'

    /*style for li elements with a class attribute where the value ends with "class"*/
    li[class$="class"] {
    color: purple;
    }

  • [attr*=val] – this will select all elements with the attribute attr for which the value contains the string 'val' (unlike [attr~=val] it does not treat spaces as value separators but as part of attribute name)

    /*style for li elements with a class attribute where the value includes "class" anywhere in the string*/
    li[class*="class"] {
    text-decoration: underline;
    }

Pseudo Class

Pseudo-classes allow you to select one or more elements that exist in a certain state such as mouse hover, or an element that is the first child of its parent in the DOM tree (like the first word in a paragraph, or the first div in an article, etc.)

A pseudo-class is a keyword preceded by a colon(:) added to the end of selectors to specify you want to style the selected elements but only when in a certain state such as mouse hover.

  • /*Change color of link on mouse hover*/
    a:hover {
    color: purple;
    }

  • Some examples...

    • :active

      ...represents an element (such as a button) that is being activated by the user. When using a mouse, "activation" typically starts when the user presses down the primary mouse button

    • :checked

      ...represents any radio, checkbox, or option type input element that is checked or toggled to an on state.

    • :default

      ...represents any radio, checkbox, option, or button type input element that hat are the default in a group of related elements.

    • :defined

      ...represents any element that has been defined (This is useful if you have a complex custom element that takes a while to load into the page — not(:defined) - you might want to hide instances of the element until definition is complete)

    • :disabled

      ...represents any disabled element.

    • :empty

      ...represents any element that has no children.

    • :enabled

      ...represents any enabled element.

    • :first

      ...represents the first page of a printed document and is used with @page rule.

    • :first-child

      ...represents the first element among a group of sibling elements.

    • :first-of-type

      ...represents the first element of its type among a group of sibling elements.

    • :focus

      ...represents an element that has received focus. It is generally triggered on form input when the user clicks or taps on an element or selects it with the keyboard's Tab key.

    • :focus-within

      ... represents an element that has received focus or contains an element that has received focus (For example used for highlighting an entire formcontainer when the user focuses on one of its input fields)

    • :host

      ... selects the shadow host of the shadow DOM containing the CSS it is used inside(has no effect when used outside a shadow DOM)

    • :host()

      ...selects the shadow host of the shadow DOM containing the CSS it is used inside BUT only if the selector given as the function's parameter matches the shadow host.

    • :hover

      ...triggered when the user hovers over an element with the cursor (mouse pointer).

    • :indeterminate

      ...represents any form element whose state is indeterminate (such as checkboxes which have their HTML indeterminate attribute set to true, radio buttons which are members of a group in which all radio buttons are unchecked, and indeterminate progress elements)

    • :in-range

      ...represents an input element whose current value is within the range limits specified by the min and max attributes (cane be used to give the user a visual indication that a field's current value is within the permitted limits).

    • :invalid

      ...represents any input or other form element whose contents fail to validate (useful to highlight errors when filling out a form).

    • :lang()

      ...matches elements based on the language they are determined to be in.

    • :last-child

      ...represents the last element among a group of sibling elements.

    • :last-of-type

      ...represents the last element of its type among a group of sibling elements.

    • :left

      ...represents all left-hand pages of a printed document and is used with @page rule.

    • :link

      ...represents an element that has not yet been visited. (matches every a, area or link element with href attribute)

    • :not()

      ...represents elements that do not match a list of selectors

    • :nth-child()

      ...matches elements based on their position in a group of siblings..for example :nth-child(3) selects the third element and :nth-child(3n) selectes every third element.

    • :nth-last-child()

      ...matches elements based on their position among a group of siblings, counting from the end (same as :nth-child() but counts in backwards direction).

    • :nth-last-of-type()

      ...matches elements of a given type, based on their position among a group of siblings, counting from the end

    • :nth-of-type()

      ...matches elements of a given type (tag), based on their position among a group of siblings

    • :only-child

      ...represents an element without any siblings

    • :only-of-type

      ...represents an element that has no siblings of the same type

    • :optional

      ...represents any input, select, or textarea element that does not have the required attribute set on it.

    • :out-of-range

      ...represents an input element whose current value is outside the range limits specified by the min and max attributes (handy for givinga visual indication that a field's current value is outside the permitted limits).

    • :read-only

      ...represents an element (such as input or textarea) that is not editable by the user

    • :read-write

      ...represents an element (such as input or textarea) that is editable by the user.

    • :required

      ...represents any input, select, or textarea element that has the required attribute set on it.

    • :right

      ...represents all right-hand pages of a printed document and is used with @page rule.

    • :root

      ...matches the root element of a tree representing the document

    • :scope

      ...represents elements that are a reference point for selectors to match against.

    • :target

      ...represents a unique element (the target element) with an id matching the URL's fragment.

    • :valid

      ...represents any input or other form element whose contents validate succesfully (useful to highlight correctly filled out fields in a form).

    • :visited

      ...represents links that the user has already visited

    • :where

      ...selects any element that can be selected by one of the selectors in that list.

      (:where(ul, ol) {...})

Pseudo Element

Pseudo-elements may be used more often than pseudo-classes and are used to target elements that do not technically exist in the HTML document in that they allow you to add content to an element in the HTML document.

Similar to pseudo-classes but are preceded with two colons (::) and are added to the end of selectors to select a certain part of the element.

You can use only one pseudo-element in a selector. It must appear after the simple selectors in the statement.

  • /*add arrow icon after all external http links*/
    [href^=http]::after {
    content: '⤴';
    }

  • Some examples...

    • ::after

      ...creates a pseudo-element that is the last child of the selected element (often used to add cosmetic content to an element with the content property).

    • ::before

      ...creates a pseudo-element that is the first child of the selected element (often used to add cosmetic content to an element with the content property).

    • ::cue

      ...matches WebVTT cues within a selected element (used to style captions and other cues in media with VTT tracks).

    • ::cue-region

      ...matches WebVTT cues within a selected element (used to style captions and other cues in media with VTT tracks).

    • ::first-letter

      ...applies styles to the first letter of the first line of a block-level element BUT only when not preceded by other content!

    • ::first-line

      ...applies styles to the first line of a block-level element.

    • ::selection

      ...applies styles to the part of a document that has been highlighted by the user (such as clicking and dragging the mouse across text)

    • ::slotted()

      ...represents any element that has been placed into a slot inside an HTML template (only works when used inside CSS placed within a shadow DOM).

Combinators

Not exactly selectors themselves but ways of combining two or more selectors in useful ways for very specific selections, such as only paragraphs that come directly after headings

CSS has several ways to select elements based on how they are related to one another:

  • Group of selectors (A, B) - Any element matching A and /or B

    /*All img elements within li elements and/or all a elements within li elements*/
    li img, li a {
    border: 2px solid red;
    }

  • Descendant selector (A B) - Any element matching B which is a descendant of an element matching A

    /*Any a elements within a li element*/
    li a {
    border: 1px dotted black;
    }

  • Child selector (A > B) - Any element matching B that is a direct child of an element matching A

    /*Any ul element within a li element*/
    li > ul {
    background-color: grey;
    }

  • Adjacent sibling selector (A + B) - Any element matching B that is the next sibling of an element matching A

    /*Any p element that comes immediately after a h1 element*/
    h1 + p {
    font-size: 1.5rem;
    }

  • General sibling selector (A ~ B) - Any element matching B that is one of the next siblings of an element matching A

    /*All p elements that come after a h1 element*/
    h1 ~ p {
    color: darkgrey;
    }

Multiple Selectors

Not separate selectors; the idea is that you can put multiple selectors on the same CSS rule, separated by commas, to apply a single set of declarations to all the elements selected by those selectors

  • /*fallback font style*/
    h1, h2, h3, h4, h5, h6, p, li, a {
    font: italic normal bold normal 1em/1.5 Helvetica, Arial, sans-serif;
    }
