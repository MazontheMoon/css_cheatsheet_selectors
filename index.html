Simple Selectors

The style is applied to one or more elements based on the type of element, it’s class, or it's id. Simple eh!

  • Elements
  • Classes
  • ID
  • Universal
Elements

The style is applied to every element of that type and is useful for applying page wide or fallback styles

/*All p elements are blue*/
p {
color: blue
}

Classes

The class selector consists of a dot (.) followed by a class name (with no spaces in the class name).
Multiple elements in the document can have the same class name and a single element can have multiple class names (separated by spaces) which makes this selector type useful when applying a style to a variety of different types of elements.

/*Add a solid black border*/
.box-outline {
border: 2px solid black;
}

ID

The ID selector consists of a hashtag (#) followed by ID name (must be unique) of a given element.
This is different to class styling as multiple html elements can use the same class but only a single element can have an id (There can be only one!)

/*styling for first box in form*/
#top-box {
background-color: grey;
border: 2px dashed black;
}

Universal

The most simple selector of them all may be the universal selector.
* is the universal selector and is rarely useful by itself but often used in combination with other selectors.

/*reset default margins*/
ul * {
margin: 0;
padding: 0;
border: 1px dashed grey;
}

Attribute Selectors

Attribute selectors match one or more elements based on their attributes/attribute values.

The generic syntax consists of square brackets [] containing an attribute name followed by optional condition to match against the value of the attribute (this makes much more sense when you see the examples below).

  • Presence and Value
  • Substring value
Presence and Value

Try to match an exact attribute or attribute value of an element. This will make much more sense when you see the examples below (Well...a bit more sense)

  • [attr] – this will select all elements with the attribute attr

    /*style all li elements with a class attribute*/
    li[class] {
    list-style: none;
    }

  • [attr=val] – this will select all elements

    /*style li elements with a class attribute with the value "classy"*/
    li[class="classy"] {
    background-color: grey;
    }

  • [attr~=val] – this will select all elements with the attribute attr but only if the value 'val' is one of a space separated list of values for attr

    /*style li elements with a class attribute with a value of “class” but only when “class” is not the only value for that attribute - say that five times fast*/
    li[class~="class"] {
    color: purple;
    }

Substring value

The next level to presence and value, this offers flexible matching in a similar fashion to regular expression (also known as RegExp-like selectors)

  • [attr|=val] – this will select elements with the attribute attr for which the value is exactly 'val' or starts with val

    /*style for all li elements with a class attribute but only if the value is "class"*/
    li[class|="class"] {
    font-size: 2rem;
    }

  • [attr^=val] – this will select all elements with the attribute attr for which the value starts with 'val'

    /*style for li elements with a class attribute where the value begins with "class"*/
    li[class^="class"] {
    color: purple;
    }

  • [attr$=val] – this will select all elements with the attribute attr for which the value ends with 'val'

    /*style for li elements with a class attribute where the value ends with "class"*/
    li[class$="class"] {
    color: purple;
    }

  • [attr*=val] – this will select all elements with the attribute attr for which the value contains the string 'val' (unlike [attr~=val] it does not treat spaces as value separators but as part of attribute name)

    /*style for li elements with a class attribute where the value includes "class" anywhere in the string*/
    li[class*="class"] {
    text-decoration: underline;
    }
